Lifecycle

single-wallet pass, exact-input discount path
1. Buy pass

Trader pays the upfront pass price. FlowPass records quota and expiry against the pool id and trader wallet.

2. Swap through router

FlowPassRouter forwards trader identity in hookData and routes the exact-input swap through PoolManager.

3. Apply discount

beforeSwap returns the discounted dynamic fee only when the pass is active and quota covers the requested input.

4. Consume quota

afterSwap deducts actual input volume. Expired or insufficient passes fall back to the base fee path.
